
=========================================================
FILE STORAGE API DESIGN - ONE PAGE CHEAT SHEET
=========================================================

PURPOSE
-------
Build a service like Google Drive, Dropbox, OneDrive where users can:

✓ Upload Files
✓ Download Files
✓ List Files
✓ Delete Files

---------------------------------------------------------
FUNCTIONAL REQUIREMENTS
---------------------------------------------------------

Upload    -> Store files in cloud
Download  -> Retrieve files
List      -> Show user uploaded files
Delete    -> Remove files

Key Points:
- Only owner can delete
- Authenticated users only
- Support large files

---------------------------------------------------------
NON-FUNCTIONAL REQUIREMENTS
---------------------------------------------------------

Reliability  -> No data loss
Security     -> Secure access
Scalability  -> Handle millions of files
Availability -> Service always accessible
Low Latency  -> Fast upload/download

Key Points:
- Replication
- Backup
- Encryption
- CDN
- Rate Limiting

---------------------------------------------------------
API ENDPOINTS
---------------------------------------------------------

UPLOAD

POST /api/v1/files

Request:
Metadata(JSON)
+
File(Binary)

Response:

{
  "fileId":"123",
  "status":"uploaded"
}

---------------------------------------------------------

LIST FILES

GET /api/v1/files

Response:

[
  {
    "id":"123",
    "name":"resume.pdf"
  }
]

---------------------------------------------------------

DOWNLOAD FILE

GET /api/v1/files/{fileId}

Response:
Actual File Content

---------------------------------------------------------

DELETE FILE

DELETE /api/v1/files/{fileId}

Response:
204 No Content

---------------------------------------------------------
UPLOAD WORKFLOW
---------------------------------------------------------

User Selects File
      |
Authentication
      |
Authorization
      |
Upload To Temp Storage
      |
Processing
      |
Compression
      |
Encryption
      |
Store In Blob Storage
      |
Save Metadata In SQL
      |
Return Success

Key Points:
→ Metadata → SQL
→ File → Blob Storage

---------------------------------------------------------
DOWNLOAD WORKFLOW
---------------------------------------------------------

Client Request
      |
Authentication
      |
Metadata Lookup
      |
Get Storage Path
      |
Fetch File
      |
CDN (if available)
      |
Return File

Key Points:
→ Use CDN for faster delivery
→ Stream large files

---------------------------------------------------------
DELETE WORKFLOW
---------------------------------------------------------

Verify Owner
      |
Delete Blob Storage Object
      |
Delete Metadata
      |
Return 204

Alternative:
→ Soft Delete
→ status = DELETED
