┌─────────────────────────────────────────────────────────────────────────────┐
│                    PAYMENT SYSTEM + STRIPE CHEAT SHEET                     │
├─────────────────────────────────────────────────────────────────────────────┤
│ PURPOSE                                                                     │
│ Enable secure online payments between Customer and Merchant.               │
│ Stripe acts as Payment Gateway + Payment Processor.                        │
├─────────────────────────────────────────────────────────────────────────────┤
│ MAIN ENTITIES                                                               │
│ Customer      → Buys product                                                │
│ Merchant      → Sells product                                               │
│ Issuer Bank   → Customer's bank                                             │
│ Acquirer Bank → Merchant's bank                                             │
│ Card Network  → Visa/MasterCard/RuPay                                      │
│ Stripe        → Routes, validates & settles payments                        │
├─────────────────────────────────────────────────────────────────────────────┤
│ REAL LIFE EXAMPLE                                                           │
│ Rajesh buys a ₹1000 headphone from Amazon using HDFC Credit Card.          │
│ Amazon uses Stripe.                                                         │
│ HDFC = Issuer Bank                                                          │
│ Amazon Bank = Acquirer Bank                                                 │
│ Visa = Card Network                                                         │
│ Stripe handles payment processing.                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│ AUTHORIZATION PHASE (2-5 seconds)                                           │
│ 1. Customer enters card details                                             │
│ 2. Merchant → Stripe                                                        │
│ 3. Stripe → Visa/MasterCard                                                 │
│ 4. Network → Issuer Bank                                                    │
│ 5. Bank checks funds                                                        │
│ 6. Funds HOLD placed                                                        │
│ 7. Authorization Code returned                                              │
│ 8. Payment Approved                                                         │
│                                                                             │
│ Result: Money NOT moved yet, only RESERVED.                                │
├─────────────────────────────────────────────────────────────────────────────┤
│ SETTLEMENT PHASE (Hours/Days)                                                │
│ 1. Stripe collects approved transactions                                    │
│ 2. Sends to Acquiring Bank                                                  │
│ 3. Network validates                                                        │
│ 4. Issuer transfers money                                                   │
│ 5. Stripe Balance updated                                                   │
│ 6. Merchant receives payout                                                 │
│                                                                             │
│ Result: Actual money transfer happens.                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│ STRIPE CORE SERVICES                                                        │
│ Customer Service → Manage customers/cards                                   │
│ Session Service  → Checkout session                                         │
│ Invoice Service  → Generate bill                                            │
│ Charge Service   → Charge card                                               │
│ Refund Service   → Return money                                              │
│ Balance Service  → Track merchant balance                                   │
│ Payout Service   → Transfer money to bank                                   │
│ Pub/Sub Service  → Async communication                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│ IMPORTANT APIS                                                              │
│ POST /customers         → Create Customer                                   │
│ POST /invoices          → Create Invoice                                    │
│ POST /customerSessions  → Start Checkout Session                            │
│ POST /charges           → Charge Card                                       │
│ POST /refunds           → Refund Payment                                    │
│ POST /payouts           → Transfer Money to Merchant Bank                   │
│ GET  /balance/{id}      → Check Balance                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│ ARCHITECTURE DECISIONS                                                      │
│ Client ↔ API Gateway      = REST API                                        │
│ API Gateway ↔ Services    = GraphQL                                         │
│ Request Format            = x-www-form-urlencoded                           │
│ Response Format           = JSON                                            │
│ Protocol                  = HTTP/2 + TLS 1.3                               │
├─────────────────────────────────────────────────────────────────────────────┤
│ SECURITY MUST-HAVES (Interview Point)                                       │
│ ✓ TLS 1.2/1.3 Encryption                                                    │
│ ✓ PCI-DSS Compliance                                                        │
│ ✓ Tokenization (Don't store card numbers)                                   │
│ ✓ API Keys / OAuth                                                          │
│ ✓ Idempotency Keys (Avoid duplicate payment)                                │
│ ✓ Fraud Detection                                                           │
│ ✓ Rate Limiting                                                             │
│ ✓ Input Validation & Sanitization                                           │
│ ✓ Audit Logs                                                                │
│ ✓ Secure File Transfer (SFTP)                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│ COMMON ERROR CODES                                                          │
│ 400 → Bad Request                                                           │
│ 401 → Unauthorized                                                          │
│ 402 → Insufficient Funds                                                    │
│ 403 → Forbidden                                                             │
│ 404 → Not Found                                                             │
│ 409 → Duplicate Request (Idempotency Conflict)                              │
│ 429 → Too Many Requests                                                     │
│ 500 → Internal Server Error                                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│ INTERVIEW ONE-LINER                                                         │
│ "Stripe receives payment requests, securely authorizes transactions         │
│ through card networks and issuer banks, manages balances, refunds,          │
│ and payouts, then settles funds to merchants."                              │
└─────────────────────────────────────────────────────────────────────────────┘
